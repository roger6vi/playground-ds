# API del Design System (placeholder)

Esta carpeta contendrá una API Express/Node que expondrá:
- Endpoints para tokens y metadatos de componentes.
- Integraciones con MCP o servicios externos.

Estructura planificada:
- `server.js`: punto de entrada que inicializa Express.
- `routes/`: define las rutas HTTP.
- `controllers/`: lógica para cada ruta.
- `models/`: acceso a datos (por ejemplo, MongoDB).

Añade la implementación cuando se desarrolle la API.
